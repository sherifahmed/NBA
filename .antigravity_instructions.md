# Workspace Instructions (Antigravity AI)

Welcome to the Neelus Boarding Adventure (NBA) workspace! When contributing to this project, you must adhere strictly to these operational rules:

## 1. Test-Driven Development (TDD) Required
All features, modifications, and fixes must follow TDD principles:
- **Red/Green/Refactor**: Write a failing unit test first in the `src/pages/__tests__/` or `src/components/__tests__/` directory.
- Verify tests fail for the right reason.
- Implement strictly what is needed to make the test pass.
- Ensure 100% test consistency using `vitest` and `react-testing-library`. Do not push changes unless tests pass.

## 2. Design Consistency & Quality
This app utilizes a unique, high-end "BBA Dog-Tech Matrix" design language. 
- **Required Context**: Before styling components, review `docs/DESIGN_STANDARDS.md` and `src/index.css`.
- Use the `premium-glass` utility to match our glassmorphism layout standard.
- Incorporate the brand identity elements (BBA Colors: Deep Azure, Copper/Orange, Silver) and the `AmbientBackground`.

## 3. Component Architecture
- Separate logic from layout. Complex graphical components (like SVGs or Ambient Backgrounds) must be externalized into `src/components/`. 
- Global context should remain in `App.tsx` utilizing minimal inline styles.

## 4. Git Flow Workflow
- Each logical unit of work must be committed to git sequentially. 
- Use standard `feat:` / `fix:` / `chore:` commit messages.
- Run `npm test` before `git push`.
