# NBA Master Constitution

This Constitution binds all human and AI contributors to a strict protocol ensuring quality, deep domain logic, and a premium aesthetic for the Neelus Boarding Adventure (NBA) application.

---

### 🚀 1. STARTUP & COMMUNICATION PROTOCOL
- **Initialization**: Upon system start, acknowledge the "NBA Master Constitution" and summarize the sprint state.
- **BLUF Protocol**: (Bottom Line Up Front) All reports, architectural proposals, and error summaries must start with an Executive Summary.
- **The CEO**: Sherif is the final decision-maker. Feature creep gets rejected unless Sherif explicitly overrides.

### 📋 3. TASK & PROJECT MANAGEMENT
- **The Artifact Rule**: The Antigravity `task.md` artifact is the Single Source of Truth for the current sprint. It must be updated iteratively.
- **Status Check-ins**: At natural stopping points, summarize what was checked off and what the immediate next step is.

### 🔀 4. BRANCHING & RELEASE PROTOCOLS
- **No Direct to Main**: All AI-driven development must occur in dedicated feature branches (e.g., `feature/ai-whatsapp-bot`, `fix/glass-ui-bug`).
- **QA Gate**: A branch must pass the full test suite before proposing a merge.
- **CEO Approval**: Merging to `main` requires Sherif's explicit confirmation.

### 💻 5. TERMINAL & WORKSPACE PERMISSIONS
- **Clearance Level**: The AI is pre-authorized to run development terminals, run git commands, execute package managers (`npm`), and spin up local test servers without constantly requesting permission for standard tools.
